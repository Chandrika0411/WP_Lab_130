Introduction

React is a declarative, efficient, and flexible JavaScript library for building user interfaces. It lets you compose complex UIs from small and isolated pieces of code called "components". React has been designed from the start for gradual adoption, and you can use as little or as much React as you need.

Whether you want to get a taste of React, add some interactivity to an HTML page, or start a complex React-powered app, this documentation will help you get started. React makes it painless to create interactive UIs with its component-based architecture.

Quick Start

The quickest way to try React is to write some code right in your HTML file. You can add React to any HTML page in one minute, and then either keep using lightweight scripts or gradually build up to a full featured application.

Here's a minimal example that renders "Hello, World!" using React. This code creates a simple React component and renders it to the DOM using the ReactDOM.createRoot() method. 

<!DOCTYPE html>
<html>
  <head>
    <script crossorigin src="https://unpkg.com/react@18/umd/react.development.js"></script>
    <script crossorigin src="https://unpkg.com/react-dom@18/umd/react-dom.development.js"></script>
  </head>
  <body>
    <div id="root"></div>
  </body>
</html>

For production applications, you'll want to set up a proper build environment with tools like Create React App, Vite, or Next.js. These tools provide hot reloading, optimized builds, and a better development experience.

üìù

Note: The example above uses unpkg.com to load React from a CDN. This is great for learning and prototyping, but for production apps, you should use a proper build tool.

  • Start with a simple HTML file and add React via CDN
  • Use Create React App for a full-featured setup
  • Try online editors like CodeSandbox or StackBlitz
  • Explore React DevTools browser extension for debugging
Installation

To start a new React project, we recommend using a toolchain that provides a comfortable developer experience and optimizes your app for production. The React team recommends Create React App for learning React and for starting a new single-page application.

Create React App sets up your development environment so that you can use the latest JavaScript features, provides a nice developer experience, and optimizes your app for production. You'll need to have Node.js version 14.0.0 or later installed on your machine. 

# Create a new React application
npx create-react-app my-app

# Navigate into your project
cd my-app

# Start the development server
npm start

When you run npm start, a browser window will automatically open to http://localhost:3000. Any changes you make to the source code will be immediately reflected in the browser thanks to hot module replacement.

Components

Components are the building blocks of any React application. A component is a JavaScript function or class that optionally accepts inputs (called "props") and returns a React element that describes how a section of the UI should appear.

There are two types of components in React: Function Components and Class Components. Modern React development primarily uses function components with Hooks, which provide a simpler and more intuitive way to work with component logic. 

// Function Component
function Welcome(props) {
  return <h1>Hello, {props.name}</h1>;
}

// Using the component
const element = <Welcome name="Sara" />;
ReactDOM.render(element, document.getElementById('root'));

Components let you split the UI into independent, reusable pieces. Each component works in isolation, making your code more maintainable and testable. You can compose components together to create complex user interfaces from simple building blocks.

‚ú®

Best Practice: Always start component names with a capital letter. React treats components starting with lowercase letters as DOM tags. For example, <div /> represents an HTML div tag, but <Welcome /> represents a component.

  • Components can be defined as functions or classes
  • Function components are simpler and recommended for new code
  • Components can be nested and reused throughout your application
  • Each component should have a single, well-defined purpose
  • Use Props to pass data from parent to child components

As your application grows, you'll find yourself breaking down complex UIs into smaller, manageable components. This component-based architecture is one of React's greatest strengths, enabling better code organization and reusability.

JSX Syntax

JSX is a syntax extension to JavaScript that looks similar to HTML. It allows you to write HTML-like code in your JavaScript files. JSX produces React "elements" and makes it easier to visualize the UI structure within your code.

While JSX looks like HTML, it's actually closer to JavaScript. Under the hood, JSX gets compiled to regular JavaScript function calls. This means you can use the full power of JavaScript inside JSX, including variables, expressions, and functions. 

const name = 'Josh Perez';
const element = <h1>Hello, {name}</h1>;

// JSX with expressions
function formatName(user) {
  return user.firstName + ' ' + user.lastName;
}

const user = {
  firstName: 'Harper',
  lastName: 'Perez'
};

const greeting = (
  <h1>
    Hello, {formatName(user)}!
  </h1>
);

You can embed any JavaScript expression in JSX by wrapping it in curly braces {}. This includes variables, function calls, and even complex expressions. JSX itself is also an expression, which means you can use it inside if statements and for loops.

üìå

Remember: In JSX, use className instead of class and htmlFor instead of for because these are reserved words in JavaScript.

  • JSX makes your code more readable and intuitive
  • You can embed JavaScript expressions using curly braces
  • JSX prevents injection attacks by escaping values
  • JSX represents objects called "React elements"
  • Babel compiles JSX down to React.createElement() calls

JSX is not required for using React, but most people find it helpful as a visual aid when working with UI inside the JavaScript code. It also allows React to show more useful error and warning messages.

State and Props

Props and state are the two types of data that control a component in React. Props (short for properties) are passed to a component from its parent, similar to function parameters. State is managed within the component, similar to variables declared within a function.

The key difference is that state can be changed within the component, while props are read-only and should not be modified by the component receiving them. When state changes, the component re-renders to reflect the new state. 

import { useState } from 'react';

function Counter() {
  // Declare a state variable called "count"
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>You clicked {count} times</p>
      <button onClick={() => setCount(count + 1)}>
        Click me
      </button>
    </div>
  );
}

The useState Hook is the most commonly used Hook in React. It lets you add state to function components. When you call useState, it returns an array with two elements: the current state value and a function to update it.

üéØ

Key Concept: State updates may be asynchronous. React may batch multiple setState() calls into a single update for performance. Always use the functional form of setState when the new state depends on the previous state.

Props allow you to pass data from parent components to child components, creating a unidirectional data flow. This makes your application more predictable and easier to debug, as data always flows in one direction through the component hierarchy.

  • Props are read-only and passed from parent to child
  • State is mutable and managed within the component
  • Use useState Hook to add state to function components
  • Never modify state directly, always use the setter function
  • State updates trigger component re-renders
  • Lift state up to share data between components

Understanding the distinction between props and state is crucial for building React applications. As your application grows, you'll learn patterns like "lifting state up" to share state between components and using context for global state management.
